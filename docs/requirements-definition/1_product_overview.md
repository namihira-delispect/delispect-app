# DELISPECT - 製品概要

## 1. 製品概要

### 1.1 製品名
DELISPECT（デリスペクト）

### 1.2 製品の目的
急性期病院の一般病棟において、AIを活用してせん妄発症リスクの高い患者を予測し、適切な予防ケアプランを提案することで、医療従事者の業務負荷を軽減しながらせん妄予防の質を向上させるWebアプリケーション。

### 1.3 背景
せん妄は身体疾患の予後不良に関連する意識障害の一種であり、患者・家族の苦痛を増し、意思決定にも悪影響を及ぼす。予防と早期介入が重要だが、現場の看護師の経験や勘に依存しており、業務負荷も高い。

## 2. システム概要

### 2.1 主要機能

- **せん妄発症予測**
  入院時の問診やバイタルなどの電子カルテ情報をインプットとして入院後2日以内のせん妄の発症率を算出する。
  インプットに必要となるデータは電子カルテ連携にて取得する。
- **看護ケアプラン作成**
  せん妄の直接因子となりうる症状を緩和するためのケア内容を提案する。
  せん妄対応経験の豊富な看護師や専門医の意見を元に作成したアプリケーションで、ケアプランの作成をサポートする。

### 2.2 運用フロー

1. 入院時に問診内容やバイタル情報、薬剤情報等を電子カルテに登録
2. 電子カルテシステムから必要な項目をインポート
3. 機械学習モデルによりせん妄発症率を「高」「低」の2段階で評価し、リスク因子を予測
4. ユーザー端末（ノートPC・タブレット）にせん妄発症率およびケアプランを表示
5. 医療従事者が提案内容を参考に看護計画を検討

### 2.3 システム構成

病院内のプライベートネットワークで動作するWebアプリケーションシステム。
同一物理サーバー上で以下を動作させる。

- **Webアプリケーション（TypeScript / Next.js）**: フロントエンド・バックエンドを統一したWebアプリケーション
- **ML API（Python）**: せん妄AI発症予測を実施するリスク判定サービス
- **RDBMS**: 患者情報等のデータを管理するデータベース

> 技術選定の詳細は [ADR-0001: Monorepo + TypeScript統一](../adr/0001-monorepo-typescript.md) を参照。

## 3. 対象ユーザー

### 3.1 プライマリーユーザー
- **看護師**: 主な利用者、せん妄予防ケアの実施
- **医師**: リスク評価結果の確認、治療方針の決定

### 3.2 セカンダリーユーザー
- **病院管理者**: システム管理、利用状況の把握
- **情報システム部門**: システム運用・保守

## 4. 制約事項

### 4.1 技術的制約
- 病院内のプライベートネットワークでの運用
- 既存の電子カルテシステムとの連携必須
- ブラウザ対応: Chrome, Edge最新版、Internet Explorer 11、EdgeHTML版Edge

### 4.2 法的・規制上の制約
- SaMD（医療機器プログラム）としての規制要件への準拠
- 個人情報保護法への準拠
- 病院の情報セキュリティポリシーへの準拠

### 4.3 運用上の制約
- 24時間365日の連続稼働
- 医療従事者のせん妄に対する教育・トレーニングが必要
